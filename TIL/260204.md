## 1. Python 학습 (Troubleshooting)

### ① SyntaxError: invalid character '“'
- **문제:** 웹이나 메신저에서 코드를 복사해오니 `SyntaxError` 발생.
- **원인:** 파이썬은 **곧은 따옴표(")**만 인식하는데, **굽은 따옴표(“)**가 들어가서 발생.
- **해결:** 에러가 난 줄의 따옴표를 지우고 키보드로 직접 다시 입력.

### ② 재귀함수 (Recursion)와 피보나치
- **개념:** 함수가 자기 자신을 계속 호출하는 구조 (마트료시카 인형).
- **핵심:** 반드시 **탈출 조건(Base Case)**이 있어야 무한루프에 빠지지 않음.

```python
def fibo(n):
    if n < 2:        # 탈출 조건 (Base Case)
        return n
    return fibo(n - 1) + fibo(n - 2) # 재귀 호출
```

- **if __name__ == "__main__":** 이 파일이 직접 실행될 때만 작동하는 테스트 코드 보호막.

## 2. Git & GitHub 실전

### ① 레포지토리 구조 정리 (폴더 이동)
- 실수로 하위 폴더에 레포지토리를 만들었을 때, `.git` 폴더를 포함해 상위로 이동시킴.
- **명령어:** `mv` (Move)

```bash
mv .git ../   # 깃 설정 폴더 이동
mv * ../      # 나머지 파일 이동
```

### ② .gitignore 설정 (필수)
- **목적:** 가상환경(`venv`), 임시파일(`__pycache__`), 보안파일(`key`, `conf`) 등을 깃허브에 올리지 않도록 차단.
- `git add` 하기 전에 가장 먼저 세팅해야 함.

### ③ 커밋 메시지 컨벤션 (규칙)
- **feat:** 새로운 기능 추가
- **fix:** 버그 수정
- **docs:** 문서(README, TIL) 수정
- **chore:** 설정 파일(.gitignore) 등 잡일
- **예시:** `git commit -m "chore: Add .gitignore file"`

## 3. 오늘의 에러 해결 (Troubleshooting)

### ① 중복/오타 파일 삭제 및 복구
- **상황:** 파일명 오타(`2620...`)로 인해 중복 파일 생성됨.
- **해결:** 로컬에서 파일 삭제 후, **반드시 깃에도 삭제 사실을 알려야 함.**

```bash
rm 잘못된파일명.py   # 1. 파일 삭제
git add .           # 2. 삭제 상태 반영
git commit -m "fix: Remove duplicate files" # 3. 확정
git push            # 4. 서버 반영
```

### ② Vim 에디터 탈출
- **상황:** `git commit` 시 메시지 없이 엔터를 쳐서 `Vim` 에디터가 켜짐.
- **해결:** `ESC` 누름 -> `:wq` 입력 -> `Enter` (저장하고 나가기).

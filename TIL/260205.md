# Docs: 26.02.05 TIL - GitHub Flow, Recursion & Git Conventions

## 1. 💡 Today I Learned (배운 내용)

### A. Git & GitHub 협업 프로세스 (GitHub Flow)
- **기본 흐름:** Issue 생성 → Branch 생성 → 작업/Commit → PR(Pull Request) → Merge → Branch 삭제.
- **이슈 연결:** PR 내용에 `resolve #이슈번호`를 적으면 Merge 시 해당 이슈가 자동 종료됨.
- **동기화:** 원격(GitHub)에서 합쳐진 후에는 로컬에서도 반드시 `git pull origin main`으로 최신화해야 함.

### B. Git 충돌(Conflict) 해결의 정석
- **공통:** 에디터에서 충돌 기호(`<<<<`, `====`, `>>>>`) 제거 및 코드 수정 → `git add .`
- **Merge 상황:** 수정 후 `git commit` 입력 (새로운 커밋 생성).
- **Rebase 상황:** 수정 후 `git rebase --continue` 입력 (다음 커밋으로 진행).

### C. 파이썬 재귀함수 (Recursion) 심화
- **구조:** `fibo(n-1) + fibo(n-2)`는 단순 덧셈이 아니라, 바닥(Base Case)까지 쪼개서 내려갔다 합쳐오는 **분할 정복(Divide and Conquer)**.
- **최적화:** **메모이제이션(Memoization)** 기법을 사용해 계산 결과를 딕셔너리에 저장(Caching)하여 중복 계산을 방지함.

### D. 개발 도구 및 마크다운
- **pre-commit:** 커밋 전 자동으로 코드를 검사해주는 도구 (설정 파일은 프로젝트 루트에 위치).
- **마크다운 팁:** 체크박스는 `- [ ]` (띄어쓰기 필수), 링크는 `[이름](주소)` 형식.

## 2. 💦 Trouble & Shooting (문제 해결)

**Q1. Git 명령어가 기억나지 않아 흐름이 끊김**
- **해결:** 무작정 암기보다 **"생성(switch -c) → 작업 → 저장(commit) → 합치기(push/merge)"**라는 행동 순서로 명령어를 매칭함.

**Q2. 마크다운 체크박스 렌더링 실패**
- **원인:** `-[]` 처럼 대괄호 안에 공백을 넣지 않음.
- **해결:** `- [ ]` 처럼 공백을 넣어 해결.

**Q3. 마크다운 링크 문법 혼동**
- **원인:** `(이름)[주소]` 인지 `[이름](주소)` 인지 헷갈림.
- **해결:** **"대괄호에는 이름, 소괄호에는 주소"** 공식으로 암기.

## 3. 📝 Cheatsheet (명령어 및 규칙 모음)

### [Git 필수 명령어]
- **브랜치 생성&이동:** `git switch -c feature/새기능`
- **작업 원격 업로드:** `git push -u origin feature/새기능`
- **로컬 동기화:** `git switch main` → `git pull origin main`
- **브랜치 삭제:** `git branch -d feature/새기능`

### [Git 커밋 컨벤션 (말머리)]
- **Feat**: 새로운 기능 추가
- **Fix**: 버그 수정
- **Docs**: 문서 수정 (README, 주석 등)
- **Style**: 코드 포맷팅 (동작 변경 없음)
- **Refactor**: 코드 구조 개선 (리팩토링)
- **Chore**: 설정 변경 및 기타 잡일

### [이슈 자동 종료 키워드]
- **Closes #번호**: 일반적인 종료
- **Fixes #번호**: 버그 수정 후 종료
- **Resolves #번호**: 요청사항 해결 후 종료
